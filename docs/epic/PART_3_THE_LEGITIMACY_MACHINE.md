## Step 4: Audit Theatre

### The Need

At some point, people will ask: "Has the code been audited?"

We need to say yes.

### The Solution

Hire an audit firm that:
- Has a professional website
- Will audit... whatever we give them
- Won't ask uncomfortable questions
- Produces a nice PDF

### What Gets Audited

| What We Claim | What We Submit |
|---------------|----------------|
| "Smart contracts audited" | A basic ERC-20 contract template |
| "Core protocol reviewed" | Documentation, not code |
| "Security verified" | The parts that exist |

### The PDF

The audit produces a PDF that says:
- "No critical vulnerabilities found in reviewed code"
- Lots of technical language
- Official-looking stamps
- Auditor's signature

**Key phrase:** "in reviewed code"

If we only gave them safe code to review, the audit is accurate. It just doesn't cover the parts that matter (which may not exist).

### Display Strategy

The PDF goes on **OUR website only.**

If anyone asks to see it on the auditor's website:
- "They're updating their site"
- "It's in their private client portal"
- "We'll send you the direct link" (never send it)
